# Step 6a Contract — S9b opencode/codex hybrid provider (AGE-251)

Orchestrator-owned interface contract for Phase 6. Derived from `proposals/s9b-AGE-251.md` (test-intent track + module map), `research/s9b-hookpoints.md` (S9a reuse symbols + host-parity obligations), and the byte-frozen `contract/v1/*.schema.json`. This is the interface Step 6b (tests) and Step 6c (code) build against. Binding manager rulings F1–F8 apply.

## Crate shape & build order (foundation-first for green compilation)

`agent-runner-opencode` is a one-shot CLI + thin lib. `src/main.rs` reads all stdin, calls `lib::handle_invocation(args, stdin) -> (stdout_bytes, exit_code)`, writes stdout, exits. Mirror S9a `src/main.rs:1-13` and `src/lib.rs:1-14` (thin facade) — opencode identity, not Claude.

Build order (each group must keep `cargo build/test --workspace` green before the next):
1. **Foundation** — `envelope`, `dispatch`, `encoding`, `schema`, `account`, `shell` (+ `describe`, `schema`, `discovery.*` wired through dispatch).
2. **Cluster A** — `terminal`, `policy`, `opencode` (event parser), `launch`.
3. **Cluster B** — `session` (+ `opencode` export parser).
4. **Cluster C** — `codex` (chatgpt-usage parser), `quota`.
5. **Cluster D** — `settings`, `setup`, `discovery` (full), `rotation`, `migration`.

Modular bar (F8): per-capability modules; every fn single-A1; declared-role header per file; `adapter_declarations` (≤5 contracts) on `opencode` + `codex` + cross-surface translators; whole-file `intrinsic_surface_declarations` on domain owners (`account`, `settings`/store, `session`/canonical). No god-file. `cargo fmt --all -- --check`, `cargo clippy --workspace --all-targets -- -D warnings`, `cargo test --workspace` all green.

## Shared types & helpers (Foundation)

- `envelope`: `CONTRACT = "oulipoly.provider/v1"`; `RequestEnvelope { contract, request_id, host, params }` (parse + validate against `common.schema.json#/$defs/RequestEnvelope:123-147`); `success_response(request_id, result) -> Value`; `error_response(request_id, category, code, message, details, retryable)` per `common.schema.json#/$defs/{SuccessResponseEnvelope,ErrorResponseEnvelope}:148-242`; `ErrorCategory` includes `conflict`, `unsupported`, `invalid_request`, `not_found`, `internal`.
- `encoding`: `encode_base64`, `decode_base64`, `sha256_hex`, `now_unix_ms`, `bounded_text`, canonical-JSON byte writer. Mirror S9a `encoding.rs`.
- `dispatch`: `subcommand_from_args`, `decode_request`, `handle_decoded_invocation`; one JSON response for non-launch; NDJSON streaming branch for `launch`; unknown/invalid subcommand → contract error envelope + nonzero exit. No `KNOWN_LATER_SUBCOMMANDS` deferral — implement the full approved surface; honest-unsupported is a real response (not a stub) for `session.replace` and `setup_brain.turn`.
- `schema`: `describe_result()` → `provider_id="opencode"`, `settings_schema_id="opencode.settings/v1"`, capabilities advertise `launch,policy,quota,session,terminal,rotation,discovery,settings,setup` true and **`setup_brain=false`** (F7); `schema_response(schema_id)` returns embedded `opencode.settings/v1`; `validate_schema_id` → error envelope on unknown.
- `account`: **ONE tested constant** (F6) — `AccountProfile { opencode_wrapper, opencode_index, codex_auth_path, codex_account_tag }` × 5:
  `opencode1`→`~/.codex/auth.json` (codex1, 781db66f); `opencode2`→`~/.codex5/auth.json` (codex5, 27f8ea6e); `opencode3`→`~/.codex2/auth.json` (codex2, 60238f0b); `opencode4`→`~/.codex3/auth.json` (codex3, 9d764739); `opencode5`→`~/.codex4/auth.json` (codex4, 835bbc4d). Quota/auth attribution follows `codex_auth_path` for the selected profile — never wrapper number. `discovery`, `settings`, `quota`, `migration` consume this constant; whole-file `intrinsic_surface_declarations`.
- `shell`: short-command runner (`run_shell_command(argv) -> {stdout, stderr, status}`), status mapping, error normalization. Mirror S9a `shell.rs` for short quota/setup commands; the launch supervisor is separate (Cluster A) and needs nested process-group/drain/cancel parity (host `supervision/*` shape).

## Per-capability contract obligations

### Foundation surfaces
- `describe` → validates `describe.schema.json#/$defs/DescribeResult`; `setup_brain=false`.
- `schema` → validates `schema.schema.json`; unknown schema_id → error envelope. **Distinguish two ids:** the contract result field `SchemaResult.schema_id` MUST equal `"opencode.settings/v1"` (the contract schema id); the embedded JSON-Schema document's own `$id` MUST be an absolute URI (or omitted) with `"$schema":"https://json-schema.org/draft/2020-12/schema"` so the embedded schema compiles **standalone** as Draft 2020-12. The embedded `$id` is NOT the relative string `"opencode.settings/v1"` (a relative `$id` without base is non-compilable). Tests assert `result.schema_id == "opencode.settings/v1"` and that `result.schema` compiles standalone — they MUST NOT assert the embedded `$id` equals the relative contract id.
- `discovery.models` → `DiscoveryModelsResult{models,warnings}`; models = `gpt-none/low/medium/high/xhigh` → `openai/gpt-5.5 --variant <effort>` (from the static alias set; no unstable opencode model-list probing).
- `discovery.accounts` → `DiscoveryAccountsResult{accounts,warnings}`; 5 accounts from the `account` constant; **no wrapper-number quota attribution** (test asserts each account's quota source = its paired `codex_auth_path`).

### Cluster A — launch / policy / terminal
- `terminal.classify` (`terminal.schema.json`): input `{stdout_base64,stderr_base64,status,observed_at_unix_ms}` → `{terminal_signal}`. **Status-derived ONLY** (F4): map `ProcessStatus` → `TerminalSignalKind` mirroring host `openai_compat.rs Recognizer::recognize` on `main`: pre-quota status wins; token predicates return `false` (no quota/rate vocabulary); post-quota maps `clean_exit`(exit 0)/`nonzero_exit`/`signal_exit`/`spawn_error`/`prolonged_silence`/`cancelled`/`unknown`. Fixtures must include quota/rate-looking strings that DO NOT change classification (host parity tests `provider_recognizer_substrings_do_not_classify_openai_compat`, `precedence_quota_text_preserves_clean_exit/nonzero_exit`).
- `policy.evaluate` (`policy.schema.json`): `{settings_id,mode,model,launch}` → `{accepted, argv, env, stdin, prompt, diagnostics, markers}`. Effective argv = selected wrapper `opencodeN` + `run --format json --dangerously-skip-permissions -m openai/gpt-5.5 --variant <effort>`; preserve `--pure` (wrapper injects it — never drop, F-forbidden); preserve env-purge/data-root via wrapper; forbidden flags/env → diagnostics. Host-prefix acceptance is an intrinsic provider launch contract, not per-account settings data: strip provider-owned prefixes whose command basename is `opencode`/`opencode1`…`opencode5` and whose managed args match either the host candidate `run --dangerously-skip-permissions -m openai/gpt-5.5 --variant <effort>` or the policy-effective `run --format json --dangerously-skip-permissions -m openai/gpt-5.5 --variant <effort>` shape. Only the remaining user-level suffix is checked for forbidden injected managed flags; selected wrapper still comes from `settings_id`→account constant.
- `launch` (`launch.schema.json`): `LaunchParams{settings_id,mode,model,argv,working_directory[,env,stdin,session]}` → NDJSON. Spawn selected `opencodeN` child (nested process group — host cancellation must not orphan opencode; mirror host `supervision/{process,drain,termination,terminal_outcome}.rs` shape). Stream: monotonic `seq`≥1, `contract`, `request_id`, `time_unix_ms`; raw child stdout/stderr bytes → `LaunchStdoutEvent.data_base64`/`LaunchStderrEvent.data_base64` (byte-preserve, do NOT transform); parse opencode `--format json` events as **metadata only** to capture `sessionID` → `LaunchMarkerEvent` + final `LaunchExitEvent.session`; heartbeat/prolonged-silence policy; final `LaunchExitEvent{status,terminal_signal[,session]}` with REAL status + status-derived signal. `opencode` module: `event_parser` pins `type`(step_start/text/step_finish)+`timestamp`+`sessionID`+nested `part` from captured fixtures without treating them as contract events.
  - Host exit-code parity (truthful `ProcessStatus` so host maps deterministically): cancelled→130, signal→128+signal, prolonged_silence→124, spawn_error/unknown→1, clean→0, nonzero→child code.

### Cluster B — session
- `session.read_turns` (`session.schema.json:41-66`): from `opencode export <sessionID>` JSON (`info`+`messages[].parts[]`); stable turn ids from message id+session id+order (not array position); `{turns,turn_count,complete}`; missing session → error envelope.
- `session.capture` (`:67-92`): extract `provider_session_id` from launch `sessionID` evidence; `{artifacts,provider_session_id,state}`; opencode source metadata, no private-DB assumptions.
- `session.export` (`:93-116`): `opencode export` native JSON → canonical bytes with `canonical_format="oulipoly.canonical_transcript/v1"` (host const), deterministic bytes, `data_base64`, `sha256`(of decoded bytes), `turn_count`(=canonical record count). MUST satisfy host `export_result_validator` (sha256 matches decoded base64; record count == turn_count).
- `session.locate_transcript` (`:22-40`): `{located:false, format_id, source_id}` — **no `path`** (opencode has no transcript file; F3). No fabricated path.
- `session.replace` (`:117-181`): **explicitly UNSUPPORTED with evidence** (F3) — return contract-valid error envelope (`unsupported`) OR `{changed:false, artifacts:[]}`; NEVER write opencode data roots/DB. Test: invoke against temp data-root with DB-like files + before/after file-hash assertion proving no mutation.

### Cluster C — quota
- `quota.source` (`quota.schema.json:8-40`): resolve paired `codex_auth_path` from account constant; `{has_source, freshness[, source_id]}`; missing/unreadable auth → `has_source:false` without calling ChatGPT.
- `quota.probe` (`:41-77`): run `chatgpt-usage <codex_auth_path>` via `shell`; parse `windows[].{used_percent,resets_at(RFC3339)}` (mirror host `quota/parse.rs` validation: reject invalid JSON/percent out 0..100/bad RFC3339); convert `remaining_ratio=(100-used_percent)/100` (range 0.0..=1.0 — host projects `1.0-remaining_ratio` back to used), `resets_at_unix_ms=epoch_ms(resets_at)`; `{available,checked_at_unix_ms,windows[{remaining_ratio,resets_at_unix_ms[,name]}]}`; nonzero script exit → truthful unavailable result.
- `quota.refresh_auth` (`:78-111`): **no-op / CLI-owned-unavailable** (F5) — `{refreshed:false, available:<probe-or-false>, checked_at_unix_ms, detail:"codex-cli-owned; agent-runner never mutates tokens"}`. NEVER mutate auth files. Test: before/after auth-file sha256 unchanged.
- `codex` module: `chatgpt_usage_parser` (adapter, ≤5 contracts) — fixture-driven raw-window parse + malformed rejection.

### Cluster D — settings / setup / discovery / rotation / migration
- `settings.*` (`settings.schema.json`): provider-owned profile store (atomic temp-config-root file; opaque versions; `conflict` on stale version; **never emit auth-token substrings**). `list/get/create/update/delete/validate/migrate` validate against schema; `validate` checks `opencode.settings/v1` semantics; `migrate` parses legacy providers.toml/model TOML → actions/warnings/diagnostics (dry_run reads only). Settings records may describe or tighten user-level profile policy, but must not gate the provider-owned host launch command shape or managed `-m`/`--variant` acceptance. `settings`/store carries `intrinsic_surface_declarations`.
- `setup.{detect,install_plan,sync_plan}` (`setup.schema.json`): detect wrappers/`opencode --version`/`chatgpt-usage`/auth/data-roots; plan objects; `setup_brain.turn` → contract-valid **unsupported** envelope (F7).
- `rotation.{assess,materialize}` (`rotation.schema.json`): requirements over settings/quota/session facts; `materialize` returns `{changed,artifacts,host_state_plan[,target_provider_session_id]}` (`RotationHostStatePlan` shape) — **no host DB/journal writes** (host applies). Materialization limited by session.replace-unsupported; document.
- `migration.{plan,apply}` (`migration.schema.json`): provider-owned migration only; plan `{actions,warnings,requires_backup[,confirmation]}`; apply writes only provider-owned artifacts `{applied_actions,artifacts,warnings,outcome}`; **no live `gpt-*`/TOML cutover** (F2 — design-only).

## Risk annotations

- `session.replace` / `quota.refresh_auth` / `session.locate_transcript`: risk is *over-implementing* — tests must prove honest unsupported/not-located + no side effects.
